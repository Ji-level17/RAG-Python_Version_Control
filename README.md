# Version-Aware RAG for FastAPI Code Generation

A retrieval-augmented generation system that generates **version-correct** FastAPI code by comparing four retrieval strategies. The pipeline ingests the FastAPI source repository, builds a versioned vector corpus, and evaluates code generation quality using RAGAS, HumanEval, and MBPP.

---

## Retrieval Strategies

| # | Strategy | Description | Latency |
|---|----------|-------------|---------|
| 1 | **Baseline Dense RAG** | Cosine similarity on dense embeddings | ~0.3s |
| 2 | **Hybrid RAG** | BM25 + dense embeddings fused with Reciprocal Rank Fusion (k=60) | ~0.5s |
| 3 | **Hierarchical RAG** | Parent-child file grouping → parent retrieval → child re-ranking | ~3.9s |
| 4 | **Import-Graph RAG** | Dense retrieval + import dependency graph expansion → re-ranking | ~10.7s |

All strategies support **version filtering**: chunks are tagged with `min_version` metadata so retrieval can target a specific Python/FastAPI version.

## Setup

### 1. Clone the repository

```bash
git clone https://github.com/Ji-level17/RAG-Python_Version_Control.git
cd RAG-Python_Version_Control
git checkout v2
```

### 2. Create a virtual environment

```bash
python -m venv .venv
# Windows
.venv\Scripts\activate
# Linux / macOS
source .venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

**Python version:** 3.10+ recommended (tested on 3.11 and 3.13).

### 4. Configure API keys

Create a `.env` file in the project root:

```env
PINECONE_API_KEY=your_pinecone_api_key
GEMINI_API_KEY=your_gemini_api_key
```

| Key | Source | Purpose |
|-----|--------|---------|
| `PINECONE_API_KEY` | [app.pinecone.io](https://app.pinecone.io) | Vector store for dense retrieval |
| `GEMINI_API_KEY` | [aistudio.google.com](https://aistudio.google.com/apikey) | Code generation + RAGAS judge (Gemini 2.5 Flash) |

## Data Preparation

### Corpus (already included)

`local_corpus.json` (695 chunks) is pre-built and included in the repository. It was generated by running `repo_processor.py` over the [FastAPI source repository](https://github.com/tiangolo/fastapi), which uses Tree-sitter AST parsing to extract Python functions and classes, then embeds each chunk with `st-codesearch-distilroberta-base` (768-dim) and upserts to Pinecone.

To rebuild from scratch:

```bash
# Step 1: Clone FastAPI source
git clone https://github.com/tiangolo/fastapi.git

# Step 2: Run ingestion (requires Pinecone)
python repo_processor.py

# Step 3: Rebuild corpus and import graph from dataset
python rebuild_corpus.py
python rebuild_import_graph.py
```

### Evaluation dataset

`ragas_dataset.json` contains 85 test records (from a 561-pair golden set split 70/15/15). Each record has: `question`, `contexts`, `ground_truth`, and `answer`.

## Running Experiments

### RAGAS Evaluation (4 strategies × 85 test cases)

```bash
python eval_rag_strategies.py \
    --gemini-key $GEMINI_API_KEY \
    --pinecone-key $PINECONE_API_KEY \
    --strategies baseline advanced hierarchical import_graph \
    --num-records 85 \
    --output-dir results_v9
```

Evaluates each strategy on four RAGAS metrics: **faithfulness**, **answer relevancy**, **context precision**, and **context recall**. Uses Gemini 2.5 Flash as both the generation model and the LLM judge.

### Retrieval Metrics (hit rate, MRR, precision, recall)

```bash
python eval_retrieval_metrics.py \
    --pinecone-key $PINECONE_API_KEY \
    --output-dir results_v9
```

Measures retrieval quality without an LLM judge, using exact and semantic matching against the golden set.

### HumanEval & MBPP Benchmarks

```bash
python eval_humaneval_mbpp.py \
    --gemini-key $GEMINI_API_KEY \
    --output-dir results_v9
```

Runs code generation on HumanEval (164 problems) and MBPP (257 problems) with pass@1 evaluation via sandboxed execution.

### Interactive Demo

```bash
streamlit run streamlit_app.py
```

Opens a browser UI at `http://localhost:8501` where you can select any of the 4 strategies, enter a FastAPI question, and see retrieved context + generated answer in real time.

## Reproducibility

- All evaluation scripts use `temperature=0` for deterministic generation.
- Random seed is fixed (`random.seed(42)`) for dataset splitting and sampling.
- Pre-computed results are stored in `results_v9/` for verification.
- The Pinecone index (`python-rag-v3`) and corpus file (`local_corpus.json`) are provided so experiments can be reproduced without re-ingesting the FastAPI repository.
